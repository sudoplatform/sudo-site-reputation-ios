Sudo Site Reputation SDK for iOS

Project setup

Clone this repo.

Install dependencies with cocoapods

pod install

Or if you’re a bundler user

bundle install && bundle exec pod install

Running integration tests

To run the integration tests,

  1. Obtain AWS credentials to download the client config from AWS parameter store.
  2. Run the following command to download the client config into the project directory.

sh download_config.sh -c <tenant_name>

  1. Select SudoSiteReputationIntegrationTests build schema and run all enabled tests.

Running unit tests

  1. Select SudoSiteReputation build schema and run all enabled tests.

Publishing

To publish a new pod version externally,

  1. Ensure the CHANGELOG.md file and podspec are updated for the release.
  2. Create a new tag on GitLab.
  3. In the release notes, copy the relevant changes from CHANGELOG.md.
  4. Run the “publish” stage of the resulting pipeline for the tag.

Consumption

In order to include this library in an iOS app:

  1. Install CocoaPods.
  2. Create a Podfile in your project root with the following
target 'MyApp' do
//...
    pod 'SudoSiteReputation’, :git => 'git@gitlab.tools.anonyome.com:platform/site-reputation/site-reputation-ios.git'
end
  1. Run pod install.
